- **السوق المستهدف:** الأردن

- **جميع ملفات المشروع مرفقة بالكامل — اقرأها سطراً بسطر قبل أي تقييم**

## التعليمات الصارمة

1. **اقرأ كل سطر** في كل ملف قبل كتابة أي ملاحظة. لا تُعمّم، لا تفترض، لا تتخيل — كل ملاحظة يجب أن تُشير لاسم الملف ورقم السطر أو اسم الدالة/الـ Selector.
2. **لا تذكر مشكلة بدون إثبات** (اقتبس الكود المُسبب أو حدد موقعه بدقة).
3. **لا تذكر مشكلة بدون حل** — أعطِ كود الإصلاح جاهزاً أو الخطوات الدقيقة.
4. **قيّم بصرامة وواقعية** — لا مجاملات. إذا كان القسم ممتازاً قل ذلك بوضوح، وإذا كان سيئاً قل ذلك بوضوح.
5. **رتّب المشاكل** داخل كل قسم حسب الخطورة: 🔴 حرج → 🟡 متوسط → 🟢 تحسين.

## أقسام المراجعة المطلوبة

### القسم 1: التصميم وتجربة المستخدم (UI/UX) — التقييم من 10
افحص:
- احترافية التصميم العام وعصريته مقارنة بمتاجر إلكترونية رائدة عربية ودولية
- تناسق الألوان (Color Palette) — هل تتبع نظرية الألوان؟ هل هناك ألوان نشاز؟
- التدرج البصري (Visual Hierarchy) — هل العين تُوجَّه للعناصر الأهم أولاً؟
- المسافات والهوامش (Spacing & Padding) — تناسق، ازدحام، فراغات زائدة
- وضوح الأزرار والـ CTAs — هل يعرف المستخدم أين ينقر؟
- جودة التأثيرات (Animations/Transitions) — هل هي سلسة أم مبالغ فيها أم ناقصة؟
- الأيقونات والخطوط — حجمها، وضوحها، اتساقها
- تباين النصوص مع الخلفيات (قِسه فعلياً بنسب WCAG)
- الوضع الداكن والفاتح — هل كل عنصر يتحول بشكل صحيح؟ هل هناك عناصر «تكسر» في أحد الوضعين؟

### القسم 2: التجاوب (Responsive Design) — التقييم من 10
افحص CSS بالكامل وحلل Media Queries:
- هل هناك breakpoints كافية ومنطقية؟
- سلوك الموقع عند: 320px, 375px, 414px, 768px, 1024px, 1440px, 1920px
- القائمة على الموبايل (hamburger) — هل تفتح/تغلق بشكل صحيح؟ هل تحبس التمرير (scroll lock)؟
- النماذج (forms) والجداول والـ modals على الشاشات الصغيرة
- الصور — هل تتكيف؟ هل هناك overflow أفقي في أي مكان؟
- النصوص — هل أحجامها مناسبة لكل شاشة؟
- أي عنصر يخرج عن حدود الشاشة أو يتداخل مع غيره

### القسم 3: الأداء والسرعة — التقييم من 10
حلّل:
- حجم كل ملف CSS/JS (الأصلي والمضغوط) — هل الفرق منطقي؟ هل الضغط فعّال؟
- كود JavaScript غير مستخدم أو ميت (Dead Code)
- CSS غير مستخدم (Unused Selectors)
- عدد طلبات HTTP — هل يمكن تقليلها؟
- هل يوجد Lazy Loading للصور والموارد الثقيلة؟
- سلوك الـ Preloader — هل يعطل التفاعل؟ هل يختفي بسرعة كافية؟
- استخدام CDN — هل الروابط صحيحة؟ هل هناك CDN واحد أم متعدد؟ هل يوجد fallback؟
- هل يوجد render-blocking resources؟
- تقدير Lighthouse Score المتوقع (Performance, LCP, FID, CLS)
- هل هناك memory leaks في JavaScript (event listeners غير منظفة، intervals لا تتوقف)؟

### القسم 4: SEO — التقييم من 10
افحص:
- `<title>` و `<meta description>` — طولهما ومحتواهما وملاءمتهما للسوق الأردني
- Open Graph و Twitter Cards — هل مكتملة؟
- هيكل العناوين (H1→H6) — ارسمه كشجرة، هل هو منطقي؟ هل H1 واحد فقط؟
- Semantic HTML — هل يستخدم `<main>`, `<section>`, `<article>`, `<nav>`, `<aside>`, `<footer>` بشكل صحيح؟
- Alt text للصور — هل وصفي ومفيد أم عام أم مفقود؟
- `lang="ar"` و `dir="rtl"` — هل مضبوطة على `<html>`؟
- Schema Markup (JSON-LD) — هل موجود؟ ما النوع المناسب (LocalBusiness, Store, Product)؟ اكتب الكود المقترح
- Canonical URL
- هل الروابط الداخلية (Anchor Navigation) صديقة لمحركات البحث؟
- ملف sitemap.xml و robots.txt — هل موجودان؟ اقترح محتواهما

### القسم 5: إمكانية الوصول (Accessibility) — التقييم من 10
افحص وفقاً لمعايير WCAG 2.1 AA:
- كل `<form>` — هل كل input له `<label>` مرتبط بـ `for`/`id`؟
- كل زر/رابط — هل له نص واضح أو `aria-label`؟
- `role` attributes — هل مستخدمة حيث يلزم؟
- التنقل بالكيبورد (Tab Order) — هل منطقي؟ هل هناك عناصر لا يمكن الوصول إليها؟
- Focus indicators (`:focus`, `:focus-visible`) — هل موجودة ومرئية؟
- Modals — هل تحبس التركيز (Focus Trap)؟ هل Escape يغلقها؟ هل `aria-modal="true"`؟
- نسب التباين — حدد أي نص/خلفية لا يحقق AA (4.5:1 للنص العادي، 3:1 للكبير)
- `aria-live` regions للمحتوى الديناميكي (إشعارات، سلة التسوق)
- Skip Navigation link

### القسم 6: الأمان — التقييم من 10
**هذا القسم حرج — كن صارماً جداً:**
- مفاتيح Supabase (anon key, URL) — هل مكشوفة في الـ Frontend؟ ما الخطر الفعلي؟ ما البديل؟
- Row Level Security (RLS) — حلّل ملفات SQL (schema.sql, fix-rls.sql) — هل السياسات كافية ومحكمة؟
- Input Validation — افحص كل حقل إدخال: هل يتم التحقق؟ هل يتم تنظيف المدخلات (sanitization)؟
- XSS — هل هناك أي مكان يتم فيه إدراج مدخلات المستخدم في DOM بدون تنظيف (innerHTML, document.write)؟ حدد السطر بدقة
- CSRF — هل هناك حماية؟ هل هي ضرورية مع Supabase Auth؟
- Authentication في لوحة الإدارة — كيف تعمل؟ هل يمكن تجاوزها؟
- هل يمكن لمستخدم عادي الوصول لـ API endpoints حساسة؟
- تشفير كلمات المرور — هل يتم في Supabase Auth أم يدوياً (افحص hash.js)؟
- Content Security Policy (CSP) headers — هل هناك توصيات؟

### القسم 7: جودة الكود — التقييم من 10
افحص كل ملف:
- **HTML:** هل هو Valid؟ هل هناك أخطاء بنيوية؟ هل Nesting صحيح؟
- **CSS:** تكرار Selectors، Properties متضاربة، !important المفرط، Custom Properties (هل تُستخدم بفعالية من variables.css؟)، تنظيم الملفات
- **JavaScript:** 
  - هل يتبع ES6+ (const/let, arrow functions, template literals, async/await, destructuring)؟
  - DRY — حدد أي كود مكرر يمكن تجميعه في دالة
  - Error Handling — هل كل Promise لها catch؟ هل try/catch مستخدم حيث يلزم؟
  - Event Listeners — هل هناك listeners مكررة أو لا تُزال؟
  - Global Scope Pollution — هل هناك متغيرات عامة كثيرة؟
  - Naming Conventions — هل الأسماء واضحة ومتسقة؟
  - التعليقات — هل كافية؟ هل دقيقة أم قديمة؟

### القسم 8: الوظائف والميزات — التقييم من 10
تتبع الكود وحلل المنطق:
- سلة التسوق: إضافة، حذف، تحديث الكمية، حساب المجموع، حفظ في localStorage/Supabase
- نماذج الاتصال والصيانة: validation، إرسال، رسائل النجاح/الخطأ
- البحث والفلترة: هل يعمل في الوقت الفعلي؟ هل يشمل كل الحقول؟
- Smooth Scrolling والتنقل بين الأقسام
- كل Modal: فتح، إغلاق، محتوى ديناميكي، إغلاق بـ Escape وبالنقر خارجه
- حفظ تفضيل الوضع الداكن/الفاتح (localStorage)
- عداد الإحصائيات (Counter Animation) — هل يعمل عند الوصول للقسم؟
- زر العودة للأعلى — هل يظهر/يختفي بشكل صحيح؟
- روابط WhatsApp والأزرار العائمة
- أي ميزة أخرى تلاحظها — هل تعمل بشكل كامل أم بها خلل؟

### القسم 9: لوحة الإدارة — التقييم من 10
افحص admin.html + admin.js + admin.css:
- آلية تسجيل الدخول والتحقق — هل آمنة؟ هل يمكن تجاوزها من DevTools؟
- CRUD للمنتجات — إضافة، عرض، تعديل، حذف — تتبع الكود كاملاً
- إدارة الطلبات — عرض، تغيير الحالة، حذف
- إدارة خدمات الصيانة
- مزامنة البيانات مع Supabase في الوقت الفعلي (Realtime Subscriptions)
- سهولة الاستخدام والتصميم — هل الإدارة سهلة لشخص غير تقني؟
- معالجة الأخطاء — ماذا يحدث عند فشل الاتصال بـ Supabase؟

### القسم 10: التوافق مع المتصفحات — التقييم من 10
حلل الكود بحثاً عن:
- CSS غير مدعوم في Safari/Firefox (backdrop-filter, :has(), container queries, etc.)
- JavaScript APIs قد لا تكون مدعومة (Intl, ResizeObserver, IntersectionObserver)
- Vendor Prefixes مفقودة (-webkit-, -moz-)
- أي كود يحتاج Polyfill
