⚠️ قواعد ذهبية لازم تلتزم فيها
═══════════════════════════════════════════════════════════════

1. اشرح كل مشكلة بلغة بسيطة جداً — كأنك تشرح لصاحب محل كمبيوتر مش مبرمج.
   لكل مشكلة استخدم هذا الأسلوب:
   - 🏪 مثال من الحياة: (شبّه المشكلة بموقف حقيقي في محل أو سوبرماركت)
   - 😰 شو بيصير: (ماذا سيعاني منه الزبون أو صاحب المحل)
   - 🔧 الحل: (خطوات واضحة مرقّمة + كود إن أمكن)

2. ركّز فقط على السلبيات والأخطاء — لا تذكر أي إيجابيات نهائياً.

3. تتبّع كل عملية من بدايتها (ضغطة الزبون) إلى نهايتها (النتيجة في قاعدة البيانات) — خطوة بخطوة.

4. افحص كل "ماذا لو؟" ممكن يحصل:
   - ماذا لو أرسل المستخدم بيانات فارغة؟
   - ماذا لو انقطع الإنترنت في نص العملية؟
   - ماذا لو أرسل بيانات خاطئة (حروف بدل أرقام)؟
   - ماذا لو فتح صفحة منتج محذوف؟
   - ماذا لو ضغط الزر مرتين بسرعة؟
   - ماذا لو حاول يتلاعب بالبيانات (يغير السعر يدوياً)؟
   - ماذا لو رصيده غير كافٍ؟
   - ماذا لو المزود الخارجي ما رد؟

📋 معلومات المشروع الكاملة
═══════════════════════════════════════════════════════════════

- **اسم المشروع**: TechZone (موقع بيع وصيانة أجهزة كمبيوتر + خدمات رقمية)
- **التقنيات**: تطبيق React حديث + Supabase (PostgreSQL + Auth + Storage + Realtime)
- **الهيكل**: Model → Service → Hook → Component
- **المصادقة**: Supabase Auth
- **قاعدة البيانات**: Supabase مع RLS
- **المحفظة**: نظام رصيد داخلي (wallet) لشراء الخدمات الرقمية
- **المزود الخارجي**: Provider API لتنفيذ طلبات الخدمات الرقمية
- **نظام الطلبات مزدوج**:
  1. طلبات منتجات فيزيائية (عبر /api/checkout — دفع عند الاستلام)
  2. طلبات خدمات رقمية (عبر /api/orders/create — خصم من المحفظة)

⚠️ **مهم جداً — لوحة التحكم الخارجية:**
لوحة التحكم هي تطبيق **Vanilla JavaScript مستقل تماماً** خارج إطار الموقع الرئيسي:
  - ملف HTML ثابت: `public/admin.html`
  - إعدادات: `public/admin-config.js` (يحتوي مفاتيح Supabase!)
  - محرك البيانات: `public/js/admin/data-engine.js` (65KB)
  - الإقلاع: `public/js/admin/bootstrap.js`
  - النواة: `public/js/admin/core.js`
  - 14 وحدة إضافية في `public/js/admin/`
  - تتصل مباشرة بـ Supabase من المتصفح بدون API routes
